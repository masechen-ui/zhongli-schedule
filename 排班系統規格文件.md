## 一、法規框架（最高優先，不可違反）

### 雙週變形工時
| 項目 | 規則 |
|------|------|
| 制度 | 雙週變形工時 |
| 雙週工時上限 | 80小時（計薪） |
| 單週工時上限 | 40小時（計薪） |
| 單日工時上限 | 8小時（計薪，系統只排到這裡） |
| 月公休天數 | 8天（30天月）或 9天（31天月），不多不少 |
| 連續上班上限 | 不超過6天 |

### 雙週週期錨點
- 基準日：**2025-07-06**
- 每14天一個週期，往前往後推算全年
- 範例：5/25-6/7、6/8-6/21、6/22-7/5、7/6-7/19…

## 二、工時計算基準

### 休息時間規則（勞基法）
| 值勤時間 | 休息時間 | 計薪工時 |
|----------|----------|----------|
| 未滿4小時 | 0小時 | 等於值勤 |
| 4小時以上，未超過9小時 | 0.5小時 | 值勤 - 0.5 |
| 超過9小時 | 1小時 | 值勤 - 1 |

### 系統預設班別
| 班別 | 值勤 | 休息 | 計薪 | 說明 |
|------|------|------|------|------|
| A班 | 8小時 | 0.5小時 | **7.5小時** | 早班，時間依各店APOC設定 |
| P班 | 8小時 | 0.5小時 | **7.5小時** | 晚班，時間依各店APOC設定 |
| O班 | 最多11小時 | 1小時 | **最多10小時** | A班變形，獨自覆蓋全天 |

> 系統只排標準班（A/P = 8小時值勤），加班由組長手動調整。

## 三、月工作天與店時數

### 月工作天計算
| 月份天數 | 公休天 | 工作天 |
|----------|--------|--------|
| 30天月 | 8天 | 22天 |
| 31天月 | 9天 | 22天 |

### 店月時數上限公式
```
店月時數上限 = (月天數 - 公休天數) × 8h × 派員數
```

| 派員數 | 30天月 | 31天月 |
|--------|--------|--------|
| 0.5 | 88h | 88h |
| 1.0 | 176h | 176h |
| 1.5 | 264h | 264h |
| 2.0 | 352h | 352h |
| 3.0 | 528h | 552h |

## 四、班別代號邏輯

### 基本原則
- A / P / O 是代號，**實際執勤時間依各店APOC設定**
- 組長保留手動修改任何格子的權力

### 各店型預設班別
| 店型 | 預設班別 | 說明 |
|------|----------|------|
| 一般專櫃 | A班 | 全天早班 |
| 巴斯 | P班 | 下午上班 |
| 百貨（1.5人以上） | A + P 搭配 | 主副互補 |

### O班發生條件
| 情境 | 說明 |
|------|------|
| 去支援其他櫃台 | 離開本店 |
| 2人店，一人休假 | 剩一人獨自覆蓋 |
| 3人店，兩人都休假 | 剩一人獨自覆蓋，最多11小時值勤 |
| 特殊狀況 | 組長判斷 |

## 六、公休分配規則

### 計算方式
```
雙週公休天 = 4天（固定）
跨月雙週公休 = 按本月天數佔整雙週比例分攤
月總公休 = 各雙週公休加總 = 8~9天
```

### 分配原則
1. 均勻分布在各雙週內
2. 多人店錯開排，同一天最多1人公休
3. 單人店可直接休（店沒人是正常的）
4. 連續上班不超過6天
5. 組長可手動調整或鎖定

## 十、已知問題與注意事項

### 排班邏輯已確認的規則
1. `estimateAvgShiftH` 固定回傳 8（不動態估算）
2. 公休只來自公休分配段，不用「空格→公休」填充
3. 連班保險只掃 `type==='work'` 的格，空格不算連班
4. `storeShiftTimes` 載入後必須同步到 `stores[].shiftTimes`
5. 休息時間：未滿4小時不休，超過9小時休1小時

### 尚未完美的地方
- 公休偶爾多1天（10天而非9天），可由組長調整
- 未排班空格偶爾出現2天，可由組長補排
