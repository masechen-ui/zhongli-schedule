## v11（2026-06-13）三視角統一呈現格式

| 項目 | 內容 |
|------|------|
| 三視角呈現方式不一致（店家/人員=每人豐富區塊；總表=緊湊一人一列） | 新增共用函式 `renderPersonBlock(s,y,m)`（店家視角格式：星期/班別/服勤時間/用餐/代班/支援/備註說明/AP班＋左側資訊欄＋右側統計） |
| 中壢課總表改版 | 由緊湊 `.sg` 表 → **依組分群、每人一個豐富區塊**（唯讀；編輯仍在店家／人員視角操作）。沿用 `margin-bottom:16px` 包裝，列印時每人不跨頁 |
| 人員視角日期對齊 | 表頭日期由「14」改為「06-14」（MM-DD），與店家視角／總表一致；星期改由獨立「星期」列顯示 |
| 未排班看不出來 | 新增 `.cell-unsched`（暖色斜紋高亮＋「未」字），人員/店家/總表三視角的 null 格都改為高亮顯示，提醒組長補排 |
| 未排班被算成公休 | 修正三視角統計：`null`（未排班）原本被併入 `offDays`（公休），改為獨立計數 `unsched`。總表公休數恢復正確 |
| 店家視角缺統計 | 右側補上「公休 X天／特休 X天／未排班 X天」（未排班>0 以警示色標示），補班時資訊完整 |
| 店家表格框線破掉 | 日期格原本無邊框，新增 `.pblock` 類別統一三視角所有格子邊框，格線完整 |
| 總表人員過濾 | 加入切換鈕：**預設只顯示「本月有實際班表」的人**，按下顯示全部；統計卡工時與人數維持全員真實總計，不受切換影響（`STATE._scheduleAllShowAll`） |

> 店家視角＝此格式來源、人員視角已為近似格式，故本次主要轉換總表。已用模擬 DOM 實跑 `renderScheduleAll()` 驗證執行期無誤。

## v9（2026-06-12）Firebase 雲端同步修復

| 問題 | 修法 |
|------|------|
| `const STATE` 無法被 Firebase `<script type="module">` 存取，`window.STATE` 永遠是 undefined | 改為 `window.STATE =`（第 338 行） |
| onSnapshot → loadFromCloud → autoSave → saveToCloud 無限循環 | 加入 `_cloudLoadingLock` 旗標，載入期間阻止 autoSave 回寫；loadFromCloud 完成後只寫 localStorage |
| Race condition：`_lastSavedAt` 在 `await setDoc` 之後才設定，onSnapshot 先收到通知無法識別為自己推的 | `_lastSavedAt` 移到 `setDoc` 之前設定；onSnapshot 比對改用 2 秒容錯視窗 |
